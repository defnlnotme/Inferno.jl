Technical Specification of the Qwen3.5 Unified Architecture for High-Performance InferenceThe advent of the Qwen3.5 family represents a transformative shift in the architecture of large-scale foundation models, transitioning from traditional transformer-based designs to a highly efficient hybrid framework. This development is characterized by the convergence of linear attention mechanisms, sparse Mixture-of-Experts (MoE), and native multimodal integration. For the purpose of implementing inference support within the Julia programming language—a language prioritized for its high-performance numerical computing capabilities—it is essential to understand the intricate interplay between these components. This report provides an exhaustive architectural decomposition of the Qwen3.5 family, encompassing both dense and MoE variants, to serve as a definitive blueprint for the development of optimized inference kernels and agentic workflows.The Evolution of the Hybrid Attention BackboneThe core architectural innovation in Qwen3.5 is the departure from a monolithic reliance on standard softmax-based self-attention. While the Transformer architecture has dominated the field due to its precise sequence modeling and parallelization benefits, its quadratic scaling with respect to sequence length presents a fundamental bottleneck for the long-context requirements of modern agentic applications. Qwen3.5 addresses this by implementing a hybrid attention core that interleaves Gated Delta Networks (GDN) with standard Gated Attention layers, typically in a 3:1 ratio.Gated Delta Networks: The Linear Recurrence MechanismThe Gated Delta Network is a linear attention variant that incorporates the delta rule for state updates, a mechanism designed to overcome the limitations of earlier state-space models (SSMs) and linear transformers. Standard linear transformers often struggle with recall and targeted memory updates, as they typically apply a uniform decay to all stored information. The GDN mechanism, however, allows the model to selectively update its internal memory, facilitating both the rapid erasure of irrelevant context and the precise retention of critical data points.For an inference agent implemented in Julia, the GDN layer must be treated as a recurrent operator during the decoding phase. The hidden state $S_t$ is updated at each time step based on the interaction between the current key ($k_t$), value ($v_t$), and the previous state ($S_{t-1}$). The "delta rule" reinterprets this recurrence as a form of online gradient descent on a reconstruction objective, where the model attempts to minimize the error between the current state and the incoming information. This is mathematically represented through a gating term $\alpha_t$ and an update strength $\beta_t$. The integration of Mamba2's gated decay mechanism further enhances this by providing data-dependent control over the state's temporal persistence.The inclusion of an output gate—an element-wise sigmoid-activated vector—is a critical stability enhancement. This gate serves to eliminate "attention sinks" and "massive activations," which are common sources of numerical instability in large-scale training and long-context inference. In Julia, this requires the implementation of a fused kernel that can handle the matrix-vector products and the element-wise gating operation in a single pass to maximize memory bandwidth utilization.Gated Attention and Numerical Stability ProtocolsEvery fourth layer in the Qwen3.5 architecture is a standard Gated Attention block. This periodic "full attention" ensures that the model maintains the high-fidelity global recall capabilities of a traditional transformer, which linear models often lack. However, even these layers are modified to incorporate Qwen's specific stability protocols.One such protocol is the application of QK-Norm, where Query ($Q$) and Key ($K$) tensors are normalized (via RMSNorm) prior to the computation of the attention scores. This prevents the exponential growth of attention logits, which is essential for maintaining precision when using FP8 or BF16 data types. Furthermore, the attention output is subjected to an additional gating mechanism derived from the layer input, ensuring that the influence of any single token is bounded.For the Julia inference engine, this means that the standard Scaled Dot-Product Attention (SDPA) kernel must be extended to support pre-attention normalization and post-attention gating. The Rotary Positional Embeddings (RoPE) are also applied differently; in the Gated Attention layers, RoPE is often restricted to a subset (typically 25%) of the hidden dimensions to improve the model's ability to extrapolate to sequences much longer than those seen during training.Mixture of Experts and High-Sparsity RoutingThe Qwen3.5 family utilizes a sparse Mixture-of-Experts (MoE) architecture for its larger variants, such as the 35B, 122B, and 397B models. This design allows the models to scale their total parameter count—and thus their "world knowledge" and reasoning capacity—without a proportional increase in the computational cost per token.Expert Segmentation and Shared Expert LogicQwen3.5 follows the "fine-grained expert" paradigm, where the traditional Feed-Forward Network (FFN) is replaced by a large number of smaller experts. Most variants utilize 256 total experts, with a "Top-8" routing strategy. A significant divergence from previous MoE implementations is the inclusion of a "Shared Expert." This expert is always active for every token, ensuring that universal linguistic patterns and common features are processed consistently, while the routed experts handle specialized tasks.In the 397B model, the total parameter count is massive, yet only 17 billion parameters are activated per forward pass. This high degree of sparsity is achieved through sophisticated routing networks that are trained to minimize load imbalance while maximizing expert specialization. For inference agents, this necessitates a "gather-scatter" approach to weight loading. In Julia, experts should be sharded across the available hardware resources (GPUs or CPU nodes), and the routing decision must be used to dynamically select and execute the appropriate expert kernels.Routing Stability and Bias MechanismsTo prevent the common MoE problem of "expert collapse"—where only a small subset of experts is ever utilized—Qwen3.5 employs a load-balancing mechanism that can be further tuned during inference. Some variants utilize a biased routing score, where a learned or dynamic bias is added to the router's logits before the Top-K selection. This ensures that even underloaded experts receive a sufficient training signal and, during inference, provides a more diverse set of processing paths for complex queries.The FFN within each expert follows the SwiGLU architecture, which is defined by three projections: $W_{gate}$, $W_{up}$, and $W_{down}$. The activation is computed as:$$\text{Output} = (W_{down} (\text{SiLU}(W_{gate} x) \odot W_{up} x))$$
This structure is more expressive than traditional ReLU or GELU-based FFNs and is a key driver of the model's reasoning performance.Native Multimodal FoundationOne of the most profound shifts in the Qwen3.5 series is its identity as a "native" multimodal model. Unlike its predecessors, which often relied on separate vision encoders connected via adapters, Qwen3.5 was trained with an early-fusion approach. This means the model processes text, images, and video tokens within the same unified hidden space from the very first layer.Visual Tokenization and PatchificationVisual inputs are converted into tokens through a vision encoder that is tightly integrated with the language backbone. The encoder typically consists of approximately 27 layers and is designed to handle high-resolution inputs through a flexible patchification scheme.Images: These are resized to fit within a predefined pixel budget (e.g., between min_pixels and max_pixels) while maintaining their original aspect ratio. They are then divided into patches, usually $16 \times 16$ or $14 \times 14$, which are projected into the model's hidden dimension.Videos: Video processing extends this into the temporal dimension. The longest_edge parameter in the video processor defines the total pixel budget across all frames ($T \times H \times W$), allowing the model to adaptively allocate resolution based on the duration of the clip.For an inference agent, this requires a multimodal pre-processing pipeline that can convert raw media files into the specific tensor format expected by the model. Julia's Images.jl and VideoIO.jl libraries can be leveraged to implement these pre-processors, ensuring that the pixel data is correctly normalized and patchified before being fed into the hybrid attention layers.Multimodal Rotary Positional Embeddings (mRoPE)To maintain spatial and temporal awareness, Qwen3.5 utilizes Multimodal RoPE (mRoPE). Unlike the 1D RoPE used in text-only models, mRoPE decomposes the positional embedding into components representing the $X$, $Y$, and $T$ (time) axes. During the embedding phase, the hidden dimension is divided into sections, each receiving a rotation corresponding to one of these axes. For example, a hidden dimension of 32 might be split into sections of  to represent the temporal and spatial positions of a visual token.Multi-Token Prediction and Inference AccelerationQwen3.5 models are trained using a Multi-Token Prediction (MTP) objective. In addition to the standard next-token prediction, the model features auxiliary prediction heads that attempt to predict tokens multiple steps into the future.Speculative Decoding via MTPThe primary benefit of MTP for inference is the enablement of native speculative decoding. An inference agent can use the MTP heads to generate a "proposal" of several future tokens in a single forward pass. These proposed tokens are then verified by the main model logic. If the proposals are accurate, the model can generate multiple tokens per iteration, significantly increasing the effective throughput, especially in latency-sensitive applications.For Julia implementations, the inference loop should be designed to handle these variable-length outputs. The MTP heads are typically located in the final stages of the model pipeline, and their weights must be loaded alongside the standard LM head if speculative decoding is to be utilized.Technical Configuration of the Qwen3.5 FamilyThe following tables summarize the critical hyperparameters for the primary members of the Qwen3.5 series. This allows it to achieve superior reasoning performance by spreading its parameter budget across more sequential processing steps.Implementation Strategy for Julia Inference AgentsTo support the Qwen3.5 family in Julia, inference agents must move beyond simple transformer loops and implement a dual-cache management system that handles both the linear recurrence of GDN and the standard KV cache of the attention layers.Memory and Cache ManagementOne of the most significant advantages of the hybrid architecture is the reduction in KV cache size. Since only 25% of the layers (the Gated Attention layers) require a KV cache, the memory footprint for long-context tasks is reduced by approximately 75% compared to a dense transformer of the same size.However, the GDN layers introduce their own state requirements. The $S_t$ matrix in the GDN recurrence must be stored for each layer and for each sequence in a batch. For a hidden dimension $d$ and head dimension $d_k$, $d_v$, the state matrix $S_t$ is of size $d_v \times d_k$. In Julia, these states should be managed as contiguous memory blocks, and specialized BLAS (Basic Linear Algebra Subprograms) kernels should be used to perform the rank-1 updates and matrix-vector multiplications that characterize the GDN recurrence.Parallelism and Distributed ServingFor the larger MoE variants, distributing the experts across multiple GPUs is a necessity. Qwen3.5 is particularly sensitive to the choice of parallelism strategy:Tensor Parallelism (TP): Best suited for sharding the non-MoE layers and the attention components. However, at high TP sizes, the synchronization overhead can become a bottleneck.Expert Parallelism (EP): Mandatory for MoE serving. By sharding experts across GPUs and using AllToAll communication, the model can maintain high throughput even when routing tokens to a large number of disparate experts.Context Parallelism (CP): Required for processing ultra-long sequences (up to 1 million tokens). This involves sharding the sequence itself across multiple devices, which requires specialized handling of the GDN recurrence across the shard boundaries.Numerical Precision and QuantizationThe Qwen3.5 architecture is designed to be "quantization-friendly." The inclusion of QK-Norm and output gating suppresses the outliers that typically make 4-bit or 8-bit quantization difficult. For Julia implementations, supporting INT8 or FP8 through libraries like QuantizedArrays.jl can provide significant speedups on modern hardware like the NVIDIA H100 or B200.Reasoning and Thinking ModesA unique feature of the Qwen3.5 family is the integration of "Thinking" and "Non-thinking" modes into a unified framework.Thinking Mode: In this mode, the model is encouraged to generate internal reasoning steps (a "Chain of Thought") before producing a final answer. This is controlled via specific prompt templates and reinforcement learning (RL) post-training that optimizes for reasoning accuracy.Non-thinking Mode: Optimized for speed and directness, suitable for simple queries or context-driven retrieval tasks.Inference agents can switch between these modes by adjusting sampling parameters—such as setting a higher temperature for thinking tasks—or by utilizing the "thinking budget" mechanism, which allows the user to specify the maximum number of reasoning steps the model should take.Summary of Architectural Constraints for ImplementationThe implementation of Qwen3.5 inference support in Julia must adhere to the following constraints to ensure architectural fidelity and performance:Dual Cache Structure: Implement a standard KV cache for Gated Attention layers and a recurrent SSM state cache for Gated DeltaNet layers.Hybrid Interleaving: Adhere strictly to the 3:1 interleaving ratio. The 0.8B model uses 24 layers (6 macro-blocks), while the 27B model uses 64 layers (16 macro-blocks).Stability Kernels: Integrate QK-Norm and output gating into all attention and linear recurrence blocks to maintain numerical stability in long contexts.Multimodal Fusion: Implement the mRoPE and vision patchification logic to handle native multimodal inputs within the main model trunk.MoE Routing: Support fine-grained experts with Top-K routing and a dedicated, always-active Shared Expert.Speculative Decoding: Leverage the Multi-Token Prediction (MTP) weights to implement proposal-and-verification loops for increased TPS.By following this specification, other LLM agents can construct high-performance inference engines that fully exploit the unique efficiencies of the Qwen3.5 family, delivering a new generation of intelligent, agentic services within the Julia ecosystem. The architectural leap taken by Qwen3.5—fusing linear attention with MoE and native multimodality—sets a new standard for efficient scale, providing the necessary depth and flexibility for the future of decentralized and autonomous AI..
