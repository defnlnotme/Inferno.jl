## 2026-04-18 - Improved CLI Line Editor Accessibility
**Learning:** Terminal line editors in Julia that wrap `stdin` and `stdout` manually must be careful with escape sequences. A common pitfall is unconditionally skipping `\e` (ESC) characters, which effectively breaks all arrow keys, function keys, and extended navigation keys (Home/End/Delete) that rely on CSI sequences. Additionally, for robust line clearing, using the ANSI escape sequence `\e[2K` is far superior to space-padding, which fails on narrow terminals.
**Action:** Always verify that escape sequences are correctly buffered and parsed in custom terminal input loops. Use `\e[2K` for line clearing. Ensure `TTYTerminal` objects use their `out_stream` for `printstyled` to avoid type errors.
